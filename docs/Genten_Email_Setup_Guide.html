What This Does

Genten sends you a daily briefing of your agent activity by email. To do that, it needs permission to send email on your behalf through your own Gmail account. This guide walks you through giving it that permission.

You will NOT be sharing your main Gmail password. Google has a special feature called "App Passwords" that creates a unique code just for Genten. This code only works for sending email — it can't access anything else in your Google account.

Once configured, Genten will send a daily briefing at the time you specify in your settings (default is 8:00 AM UTC). The briefing includes:

  • Which agents were active in the last 24 hours
  • Total API calls and spending by agent
  • Any errors or warnings that need your attention
  • Tasks waiting for your review or approval

3

Turn On 2-Step Verification

Google requires 2-Step Verification (also called 2FA) to be turned on before it will let you generate an App Password. This is a Google security rule, not a Genten one.

Already have 2-Step Verification turned on? Follow the verification steps below to confirm it's active before continuing to Section 4. This prevents errors at the App Passwords page.
Check If 2-Step Verification Is Active
→
Go to your Google Account Security page
https://myaccount.google.com/security
→
Scroll to "How you sign in to Google"
→
Look for "2-Step Verification"
If it says "On" or shows your phone number, you're all set — skip to Section 4. If it says "Off", continue below.
Turn On 2-Step Verification (If Not Already Enabled)
→
Click "2-Step Verification"
→
Click "Get Started"
→
Follow Google's prompts to set it up
Google will typically ask for your phone number and send you a verification code. This usually takes 2-3 minutes.
2-Step Verification is now active. You're ready to generate an App Password in the next section.
4

Generate an App Password

An App Password is a 16-character code that acts like a temporary password, but only for one specific app — in this case, Genten. It doesn't give access to anything else in your Google account.

→
Go to Google's App Passwords page
https://myaccount.google.com/apppasswords
→
If Google asks you to sign in again, do so
This is normal for security-sensitive pages.
→
In the "App name" field, type: Genten
→
Click "Create"
→
Google will show you a 16-character code
It will look like this: abcd efgh ijkl mnop
→
Copy this code — you'll need it in the next section
Save this code somewhere safe right now. Google will NOT show it to you again after closing this window. If you lose it, you'll have to generate a new one. A password manager is a great place to store it.
Common Error

Problem: "I don't see an 'App Passwords' option."

Solution: This usually means 2-Step Verification isn't turned on yet. Go back to Section 3 and enable it first. After enabling 2FA, wait a minute and then try accessing the App Passwords page again.

5

Add Your App Password to Genten

Now we need to tell Genten about your Gmail account and the App Password you just created. This is done by editing the .env file in the folder where you installed Genten.

→
Find the folder where you installed Genten
This is the folder that contains your .env file and docker-compose.yml file (if you're using Docker).
→
Open the .env file in a text editor
On Mac: TextEdit. On Windows: Notepad. Remember to show hidden files if you don't see the .env file — press Cmd+Shift+. in Finder (Mac) or enable "Show hidden files" in File Explorer (Windows).
→
Find the email configuration section
It will have these lines (or you'll need to add them):
GMAIL_APP_PASSWORD=
BRIEFING_RECIPIENT=
Fill In Your Email Settings
GMAIL_APP_PASSWORD=

Paste the 16-character App Password from Section 4 here. You can remove the spaces or leave them — both work. No quotes needed.

Example: GMAIL_APP_PASSWORD=abcdefghijklmnop

BRIEFING_RECIPIENT=

The email address where you want to receive the daily briefing. This can be your Gmail address or any other email you prefer.

Example: BRIEFING_RECIPIENT=you@example.com

The .env file doesn't use quotes around values. Just paste them in directly after the equals sign.
→
Save the .env file
Email settings saved. Now you need to restart Genten for the changes to take effect.
6

Restart Genten to Apply Changes

Genten reads the .env file when it starts up, so you need to restart it for the new email settings to take effect.

If You're Using Docker
→
Open Terminal (Mac) or PowerShell (Windows)
→
Navigate to your genten folder
Use cd to change directories. For example: cd ~/Desktop/genten
→
Stop Genten
If Genten is running in the foreground, press Ctrl+C. If it's running in detached mode, run:
docker compose down
→
Start Genten again
docker compose up
If You're Using npm
→
Stop the server with Ctrl+C in the terminal where Genten is running
→
Restart it
npm run dev
What to Look For

Watch the logs as Genten starts. You should see a message that looks like: 

[mailer] Email briefings enabled
[mailer] Briefing recipient: you@example.com
Email settings loaded successfully. Genten is now ready to send daily briefings.
See an error about Gmail? This usually means the App Password was pasted incorrectly. Double-check that there are no extra spaces or quotes in your .env file and restart Genten again.

Test email received! Your email setup is working correctly. Genten will now send daily briefings at your configured time (default is 8:00 AM UTC).
Common Problems

No email arrives after 2 minutes:

  • Check your spam folder — first-time emails from Genten may land there
  • Check Genten's logs for error messages about Gmail authentication
  • Verify the BRIEFING_RECIPIENT email is correct in your .env file

Error: "Username and Password not accepted"

The App Password was typed incorrectly. Generate a new App Password in Section 4 and update your .env file.

Error: "Invalid login"

The App Password doesn't match the Gmail account. Make sure you're using the same Gmail account for both generating the App Password and the BRIEFING_RECIPIENT setting.

8

Common Problems and Fixes

Quick solutions to the most common email setup issues.

I forgot to save my App Password and closed the window

Solution: Generate a new one. Go back to Section 4, visit the App Passwords page, and create a new App Password called "Genten" (or "Genten 2" if you want to keep track). The old one won't work anymore once you create a new one.

Emails land in my spam folder

Solution: This is normal for first-time emails from a new sender. Mark the first briefing email as "Not Spam" to train your email filter. Future briefings should go to your inbox.

I want to turn off email briefings

Solution: Open your .env file, remove the GMAIL_APP_PASSWORD= line (or leave it blank), and restart Genten. Briefings will still be stored in your dashboard but won't be emailed.

I want to use a different email provider (not Gmail)

Solution: Currently Genten only supports Gmail SMTP for email briefings. Support for other providers (like Resend) is planned for v1.1.

I want to change the time briefings are sent

Solution: Add this line to your .env file and restart Genten: 

BRIEFING_HOUR=14

The number is the hour in UTC (0-23). For example, 14 = 2:00 PM UTC. To convert your local time to UTC, check timeanddate.com.

Security Notes

How App Passwords work and what they can (and can't) do.

App Passwords only work for the one app you named them for. Someone with this code cannot log into your Google account, read your email, or access your other Google data. It's strictly for sending email on behalf of the app you authorized — in this case, Genten.
Revoking Access

If you want to revoke Genten's access to send email at any time:

→
Go to myaccount.google.com/apppasswords
→
Find the "Genten" entry in the list
→
Click the trash icon to delete it

Genten will stop being able to send emails immediately. You'll see errors in the Genten logs the next time it tries to send a briefing.

Where the App Password Is Stored

The App Password is stored in your .env file on your own computer. It never leaves your machine. Genten doesn't send it to any external service — it only uses it to authenticate with Gmail's SMTP servers when sending your briefings.

Your email is secure. App Passwords are designed to be safer than sharing your real Gmail password. They can be revoked at any time without affecting your main account.
